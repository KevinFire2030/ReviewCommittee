# 특허심의회 핵심 아이디어(청구항) 5분 설명 원페이지 요약

## 1. 발명의 명칭

**온디바이스 인공지능 에이전트를 이용한 자연어 기반 자기 단말 제어 장치 및 방법**

## 2. 한 문장 요약

본 발명은 **동일 스마트폰 내부에서 실행되는 온디바이스 AI 에이전트가 사용자의 자연어 명령을 해석하고, 무선 ADB 또는 로컬 브리지 기반 제어 경로를 통해 자기 단말의 앱, 운영체제 기능 및 시스템 설정을 직접 제어한 뒤, 실행 결과를 검증하고 실패 시 대체 경로로 재시도하는 자기 단말 제어 기술**이다.

## 3. 핵심 아이디어

기존 모바일 자동화나 원격 제어 기술은 대체로 외부 PC, 서버, 클라우드 환경 또는 별도 제어 주체가 대상 단말을 제어하는 구조이다. 반면 본 발명은 사용자가 실제로 보유한 스마트폰 내부에 AI 에이전트를 내재화하고, 그 에이전트가 같은 스마트폰을 직접 제어한다.

핵심은 ADB 자체가 아니라 **에이전트의 실행 위치와 제어 주체/대상 관계**이다. 즉, 본 발명은 "외부에서 단말을 제어"하는 기술이 아니라 **"스마트폰 안의 에이전트가 자기 스마트폰을 제어"하는 온디바이스 자기제어 아키텍처**이다.

## 4. 청구항 중심 핵심 구성

1. **자연어 명령 수신 및 해석**
   - 사용자의 자연어 명령을 수신하고, 수행할 작업과 실행 파라미터를 추출한다.

2. **동일 모바일 단말 내부의 AI 에이전트 실행**
   - AI 에이전트가 외부 PC나 서버가 아니라 제어 대상 스마트폰 내부에서 실행된다.

3. **단말 동일성 판단**
   - 에이전트가 실행되는 단말과 제어 대상 단말이 동일한지 판단하여 자기 단말 제어 구조를 형성한다.

4. **무선 ADB 또는 로컬 브리지 기반 제어**
   - 온디바이스 에이전트가 무선 ADB 또는 로컬 브리지를 통해 앱 실행, 화면 조작, 시스템 설정 변경 등 단말 기능을 제어한다.

5. **결과 검증 및 대체 경로 재시도**
   - 실행 후 목표 상태 도달 여부를 검증하고, 실패 시 단말 상태와 실패 원인에 따라 다른 실행 경로를 선택해 재시도한다.

6. **민감 작업 보호**
   - 결제, 삭제, 권한 변경 등 민감 작업은 사용자 확인 또는 정책 판단을 거쳐 수행한다.

## 5. 선행기술과의 핵심 차이

| 구분 | 선행기술 조사상 선행기술 | 본 발명 |
|---|---|---|
| 제어 구조 | 외부 제어 환경이 클라우드폰 또는 대상 단말을 제어 | 동일 스마트폰 내부 에이전트가 자기 단말을 제어 |
| 에이전트 실행 위치 | MCP 서버, 컴퓨팅 리소스 호출, 클라우드폰 연결 등 외부/분리 구조로 해석 | 제어 대상 스마트폰 내부에서 온디바이스 실행 |
| ADB의 의미 | 클라우드폰 또는 대상 단말에 연결하기 위한 외부 제어 수단 | 온디바이스 에이전트와 자기 단말 제어 실행부를 잇는 내부 브리지 |
| 제어 대상 | 클라우드폰 또는 외부 대상 단말 | 에이전트가 실행되는 동일 모바일 단말 |
| 기술 효과 | 대상 단말 자동화 또는 원격 제어 | 외부 PC 없는 단말 단독 자연어 제어, 지연/네트워크 의존성 감소, 개인정보 보호, 실행 신뢰성 향상 |

선행기술 조사 보고서에는 `通过ADB连接云手机`, `配置 MCP 服务器`, `调用所述计算资源以执行操作`, `클라우드폰이 ADB를 통해 연결` 등의 표현이 확인된다. 이는 선행기술이 동일 스마트폰 내부의 에이전트가 자기 단말을 제어하는 구조라기보다, 외부 제어 환경 또는 별도 컴퓨팅 리소스가 대상 단말을 제어하는 구조로 해석될 수 있음을 보여준다.

따라서 심의회에서는 다음 문장을 중심으로 설명한다.

> 선행기술과 본 발명은 ADB 또는 자동화 요소가 일부 유사해 보일 수 있으나, 본질적 차이는 에이전트가 어디서 실행되는가입니다. 선행기술은 외부 제어 환경이 클라우드폰 또는 대상 단말을 제어하는 구조이고, 본 발명은 스마트폰 내부의 온디바이스 AI 에이전트가 동일 단말을 자기 제어하는 구조입니다.

## 6. 5분 발표 흐름

### 0:00-0:40 문제 정의

기존 모바일 자동화는 외부 PC, 서버, 클라우드폰, 매크로 또는 접근성 도구에 의존하는 경우가 많다. 이 경우 실제 사용자의 스마트폰 내부에서 자연어 해석부터 실행, 검증까지 단독으로 수행하는 자기제어 구조가 부족하다.

### 0:40-1:40 발명의 핵심

본 발명은 스마트폰 내부에 AI 에이전트를 실행시키고, 사용자의 자연어 명령을 해석하여 같은 스마트폰의 앱과 시스템 기능을 제어한다. 외부에서 단말을 원격 제어하는 것이 아니라, 단말 내부 에이전트가 자기 단말을 제어하는 구조라는 점이 핵심이다.

### 1:40-2:50 청구항 요지

청구항의 핵심은 자연어 해석, 온디바이스 AI 에이전트 실행, 단말 동일성 판단, 무선 ADB/로컬 브리지 기반 자기 단말 제어, 결과 검증, 실패 시 대체 경로 재시도, 민감 작업 확인으로 정리할 수 있다. 이 구성들이 결합되어 외부 PC 없이 단말 단독으로 동작하는 폐루프 자기제어 구조를 만든다.

### 2:50-4:10 선행기술 대비 차이

선행기술 조사에서는 ADB, 자연어 처리, 작업 계획, 실행 루프가 있다는 이유로 유사성이 지적되었다. 그러나 선행기술은 클라우드폰 연결, MCP 서버 구성, 컴퓨팅 리소스 호출 등 외부 제어 환경 중심으로 해석된다. 본 발명은 동일 스마트폰 내부에서 실행되는 에이전트가 자기 단말을 제어한다는 점에서 제어 주체, 제어 대상, 실행 위치가 다르다.

### 4:10-5:00 결론

본 발명은 단순한 ADB 자동화가 아니라, 온디바이스 AI 에이전트 기반 자기 단말 제어 아키텍처이다. 선행기술과의 차이는 ADB 사용 여부가 아니라 에이전트의 실행 위치와 자기 단말 제어 구조에 있으며, 이 구조를 통해 단말 독립 실행, 개인정보 보호, 지연 감소, 결과 검증 기반 신뢰성 향상 효과를 얻을 수 있다.

## 7. 예상 질의 핵심 답변

### Q1. 선행기술에도 ADB가 있는데 무엇이 다른가?

ADB는 공지 도구일 수 있다. 다만 본 발명의 차별점은 ADB 자체가 아니라, **동일 스마트폰 내부에서 실행되는 AI 에이전트가 무선 ADB 또는 로컬 브리지를 자기 단말 제어 경로로 활용한다는 구조**이다.

### Q2. 선행기술의 클라우드폰도 모바일 단말 아닌가?

모바일 단말이라는 범주만 같다고 동일 기술은 아니다. 선행기술의 클라우드폰은 외부 제어 환경에서 연결·조작되는 대상에 가깝고, 본 발명은 사용자의 스마트폰 내부 에이전트가 동일 스마트폰을 직접 제어한다.

### Q3. 신규성/진보성 포인트를 하나만 말하면 무엇인가?

**온디바이스 AI 에이전트에 의한 동일 모바일 단말의 자기제어 구조**이다. 즉, 에이전트 실행 단말과 제어 대상 단말이 같고, 그 안에서 자연어 해석부터 실행, 검증, 대체 경로 재시도까지 수행한다는 점이다.

### Q4. 단순 자동화나 음성비서와 다른가?

단순 자동화는 정해진 스크립트나 반복 동작 중심이고, 일반 음성비서는 실제 시스템 제어 범위가 제한적이다. 본 발명은 자연어 해석, 실행 경로 선택, 무선 ADB/로컬 브리지 제어, 결과 검증, 실패 시 재시도를 결합한 폐루프 자기 단말 제어 구조이다.
